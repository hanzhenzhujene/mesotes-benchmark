This repository uses a split license model so that software and benchmark data are governed clearly and separately.

## 1. Software and tooling

Unless otherwise noted, the following are released under the MIT License:

- Python source code in `src/`
- CLI scripts in `scripts/`
- workflow and configuration files
- software-oriented project documentation

Canonical license text: [LICENSE](LICENSE)

SPDX identifier: `MIT`

## 2. Dataset artifacts

Released benchmark data artifacts are distributed under Creative Commons Attribution 4.0 International:

- JSONL benchmark records in `data/`
- schema artifacts intended as part of the released benchmark package
- other released dataset materials unless a file states otherwise

Canonical license text: [LICENSE-data](LICENSE-data)

SPDX identifier: `CC-BY-4.0`

## 3. Practical interpretation

In short:

- you may reuse and adapt the code under MIT
- you may share and adapt the released data under CC BY 4.0 with attribution

## 4. Attribution guidance

If you use MESOTES data or code in research outputs, please:

- cite the repository via [CITATION.cff](CITATION.cff)
- retain license notices where required
- clearly distinguish your modifications from the released materials

## 5. Third-party materials

If future versions of this repository include third-party content under different terms, those files should be clearly marked in place.

## 6. Priority of file-level notices

If a specific file or subdirectory includes a more specific license notice, that file-level notice controls for that material.
